# Ejercicio 1: Análisis de Complejidad con Listas y Condiciones
## Descripción
En este ejercicio, analizarás un algoritmo que:

Procesa una lista de datos.
Realiza transformaciones en cada elemento.
Ejecuta verificaciones condicionales sobre los valores generados.
El objetivo es calcular la complejidad temporal utilizando notación Big-O y justificar cómo afecta el rendimiento en diferentes casos.

### Preguntas
Cuál es la complejidad temporal del código en el peor caso?
Si la función que ejecuta la acción tuviera una complejidad O(n²), ¿cómo afectaría el rendimiento total?
¿Es posible optimizar el código para reducir su complejidad? ¿Cómo lo harías?

# Ejercicio 2: Algoritmo con Recursión y Matriz (Complejidad O(2ⁿ))
## Descripción
Este ejercicio analiza un algoritmo que:

Manipula una matriz n x n.
Ejecuta una recursión no optimizada.
Busca valores dentro de una lista.
El objetivo es calcular la complejidad temporal en el peor caso y proponer mejoras.

### Preguntas
¿Cuál es la complejidad temporal del código en el peor caso?
¿Por qué la recursión en este algoritmo tiene una complejidad de O(2ⁿ)?
Si optimizamos la recursión con memoization, ¿cómo cambiaría la complejidad?
¿Cómo afectaría el rendimiento si la ejecución de la acción tuviera una complejidad O(n²)?
